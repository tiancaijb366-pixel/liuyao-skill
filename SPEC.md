# 六爻占卜解读 Skill

## Problem Statement

用户（占卜者）提供六爻卦象数据（本卦/变卦/六亲/六神/世应/动爻/日月建），希望获得有据可查的六爻解读。现有的解读方式存在以下问题：

- **无结构化流程**：解读质量完全依赖 agent 对六爻的零散理解，不同 agent 解读结果差异大
- **缺乏系统框架**：没有融合三本核心参考书（《增删卜易》《易冒》《古筮真诠》）的方法论
- **缺少量化标准**：吉凶判断停留在感性层面，没有可追溯的量化依据
- **无验证机制**：解读结果无法与经典卦例对照验证

## Solution

开发一个结构化的六爻占卜解读 skill（SKILL.md），提供：

1. **三书融合的解读框架**：以《易冒》的分层下钻为主线（先卦后爻），嵌入《增删卜易》的四处生克冲合，和《古筮真诠》的实证对轨与心态卦理论
2. **11 张核心规则表**：纳甲、六亲、六神、八宫卦序、旺相休囚死、生旺墓绝、冲合刑害、旬空、进神退神、用神分类、应期规则
3. **量化吉凶体系**：用神十八法作为吉凶轻重量化标准
4. **测试套件**：从三本书中提取卦例，验证 agent 解读与原作一致性

## User Stories

1. 作为占卜者，我提供卦象数据后，希望能直接看到核心结论（能成/不能成/有条件），无需等待冗长的技术分析
2. 作为占卜者，我追问具体细节时，希望能获得有针对性的展开（如用神分析、动爻解读、应期判断），而不是千篇一律的模板
3. 作为占卜者，如果我忘了说明所问何事，希望 agent 能从卦象中读出某些特征来询问我，而不是直接让我重来
4. 作为占卜者，我希望解读有据可查，能追溯到具体参考书的章节
5. 作为占卜者，我希望解读能区分近病/久病、求财/守财等不同子类型的差异
6. 作为占卜者，我关心的是否是心态问题（担心/忧患），希望 agent 能识别并走心态卦路线
7. 作为 skill 开发者，我希望解读流程结构清晰，便于维护和升级
8. 作为 skill 维护者，我希望有测试套件来确保修改不破坏解读质量
9. 作为高级用户，我希望看到内部分析文件（.scratch/）中的完整推理过程
10. 作为初学者，我能通过规则表附录快速查阅六爻基础知识，而不需要翻厚书
11. 作为六爻研究者，我能通过隐式引用追溯到解读依据的具体出处
12. 作为占卜者，我希望解读后能获得可行动的建议（趋吉避凶），而不是只有吉凶判断

## Implementation Decisions

### 1. 文件结构

```
liuyao-skill/
├── SKILL.md              ← 主 skill 文件（流程 + 规则表附录）
├── CONTEXT.md            ← 术语表（已有）
├── README.md             ← 多 agent 使用说明（已有）
├── references/           ← 参考书全文
│   ├── 增删卜易.md
│   └── 易冒.md
├── docs/agents/          ← 工程配置（已有）
├── .scratch/             ← 运行时：内部分析 + 卦例测试
│   └── liuyao-skill/
│       ├── analysis/     ← agent 内部完整分析输出
│       └── tests/        ← 卦例测试用例
└── package.json          ← pi 包发布（P2 任务）
```

### 2. 内部解读流程（易冒骨架 + 三书融合）

```
① 解析数据（纯技术，不需用户）
   识别本卦/变卦/六亲/六神/世应/动爻/日月建
   ↓
② 辨卦（八种卦用）—— 易冒
   内外 / 冲合 / 反伏 / 墓绝 / 归游 / 卦候 / 卦义 / 卦验
   ↓
③ 实证对轨 —— 古筮真诠 ← 暂停点，问用户
   读出现状信息 → 让用户确认
   落实：来意核实 / 事态卦 vs 心态卦 / 真假用神 / 背景细节
   ↓
④ 辨爻（六种爻用）—— 易冒
   世应 / 两身 / 六亲 / 六神 / 爻位 / 变互
   ↓
⑤ 辨卦种 —— 古筮真诠
   静卦？→ 静卦特殊公式
   动卦？→ 正常路线（独发/多动/全动各有侧重）
   ↓
⑥ 世应十忌 —— 易冒
   空/破/绝/散/墓/动/合冲/随墓/助伤/升降进退
   ↓
⑦ 定用神 + 四处生克冲合
   事态卦 → 六亲取用
   心态卦 → 子孙(喜神) vs 官鬼(忧神) 双核心
   元神/忌神/仇神
   月建→用神 / 日辰→用神 / 动爻→用神 / 变爻回头
   ↓
⑧ 专项检查
   察动变 / 搜飞伏 / 定日月（旺相休囚死+长生十二法）
   合冲 / 旬空月破 / 刑害局化 / 六神星煞 / 暗动系统理论
   【贯穿两个层面：吉凶判断层严谨 / 应期细节层灵活】
   ↓
⑨ 用神十八法（量化评分）—— 易冒
   全吉8象 → 半吉3象 → 凶陷7象
   ↓
⑩ 应期（克期之法 + 应期公式）
   ↓
⑪ 趋吉避凶（建议）
```

### 3. 对外输出策略

- **Phase 1**（唯一自动输出）：核心结论 + 卦象总览 → **立刻停下**
- **追问驱动**：无 Phase 2/3/4，用户问什么答什么
- **内部分析文件**：写入 `.scratch/liuyao-skill/analysis/{timestamp}-analysis.md`，完整记录 11 步推理

### 4. 规则表附录（SKILL.md 末尾）

11 张表全部从参考书提取，大表分拆子表，以《易冒》为主、《增删卜易》补充：

| # | 规则表 | 来源 | 格式 |
|---|--------|------|------|
| 1 | 浑天甲子（纳甲）表 | 易冒纳甲章第四 | 8卦×6爻子表 |
| 2 | 六亲排法 | 易冒六亲章第六 | 速查行表 |
| 3 | 六神配法（日干） | 易冒六神章第七 | 10日干×6神表 |
| 4 | 八宫六十四卦卦序 | 易冒成卦章第三 | 8宫各8卦子表 |
| 5 | 五行旺相休囚死 | 增删四时旺相章 | 四季×5行表 |
| 6 | 生旺墓绝十二宫 | 易冒墓绝章第十八 | 5行×12宫子表 |
| 7 | 地支冲合刑害 | 易冒合冲章/刑害章 | 分冲/合/刑/害各子表 |
| 8 | 旬空表 | 易冒旬空章第二十六 | 10旬×2空表 |
| 9 | 进神退神规则 | 增删进神退神章 | 规则速查表 |
| 10 | 用神分类大全（细化） | 增删用神章 | 扩展分类表 |
| 11 | 应期规则大全（细化） | 易冒克期+增删应期章 | 条件×应期表 |

### 5. 两个分析层面的贯穿

吉凶判断层面严格遵循规则，应期细节层面可灵活取象：

- 生旺墓绝：吉凶层只用"两墓二绝一生旺"，细节层全面展开
- 暗动：吉凶层严格限于旺相受冲，细节层可放宽
- 刑害：吉凶层不存在，细节层可用
- 月墓月绊：吉凶层不存在，细节层可用

## Testing Decisions

### 测试策略

以**端到端集成测试**为主，辅以**中间分析验证**。

### 测试用例来源

从三本书中提取结构化卦例：

| 来源 | 提取数量 | 选取标准 |
|------|---------|---------|
| 《增删卜易》 | 15-20 例 | 覆盖各门类（占财、占病、占功名、占出行等） |
| 《易冒》 | 5-8 例 | 覆盖特色理论（用神十八法、八种卦用等） |
| 《古筮真诠》 | 5-8 例 | 覆盖心态卦、静卦、真假用神等 |

总计约 25-36 例。

### 测试用例格式

```json
{
  "id": "zs-001",
  "来源": "增删卜易·用神章",
  "问题": "占父近病",
  "时间": "辰月戊申日",
  "本卦": "乾为天",
  "变卦": "风天小畜",
  "世应": "世在上爻 应在三爻",
  "动爻": "午火动化未土",
  "日月建": "辰月 戊申日",
  "野鹤断语": "近病逢冲即愈",
  "野鹤用神": "辰土父母",
  "应期": "丑日",
  "验证点": ["用神选取", "六冲断法", "贪合忘克", "应期判断"]
}
```

### 验证指标

| 指标 | 目标 | 验证方法 |
|------|------|---------|
| 吉凶结论一致率 | ≥80% | agent Phase 1 结论 vs 野鹤/程良玉/朱辰彬原断 |
| 用神选取正确率 | ≥85% | agent 用神 vs 原作用神 |
| 应期范围匹配率 | ≥75% | agent 应期 vs 原作应期（允许 ±1 个时间单位误差） |
| 关键因素检出率 | ≥80% | agent 是否覆盖原作提到的关键因素（如冲合、空破等） |

### 测试执行

1. 每个测试用例运行 agent 一次（使用当前 SKILL.md）
2. 检查 `.scratch/liuyao-skill/analysis/` 中的内部分析文件
3. 逐项比对：结论、用神、关键因素、应期
4. 记录通过/失败及失败原因
5. 每次 SKILL.md 更新后重跑测试套件

## Out of Scope

- **排盘**：不实现排盘功能（生成卦象），只解读用户提供的已有卦象数据
- **其他占卜术**：不做八字、奇门、风水等其他术数的解读
- **GUI/Web 界面**：不提供站外 UI，保持 skill 形态
- **API 封装**：不做 pi extension 或对外 API
- **多语言**：仅中文
