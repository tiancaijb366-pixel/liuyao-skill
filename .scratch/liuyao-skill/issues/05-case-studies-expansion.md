# 05 — 测例扩充（三书全部卦例）

**What to build:** 将三本参考书（古筮真诠、增删卜易、易冒）中的全部卦例提取为结构化 JSON 测例，统一字段命名，扩充测试覆盖。

**Blocked by:** 无（可独立于 06 进行）

**Status:** ready-for-agent

## Problem Statement

现有 25 个测例（zs-* 17 例、ym-* 8 例）仅覆盖三本参考书的部分内容。《古筮真诠》虽有约 800 处卦例标记，但缺参考文件和测例。增删卜易、易冒也还有大量卦例未提取。

## Solution

按顺序手工提取三本书的全部卦例，统一为 JSON 格式，挂在 `scripts/case-studies/` 下。

## 具体内容

### 1. 字段统一

现有字段 `野鹤断语`/`作者断语` 统一为 `断语`，`野鹤用神`/`作者用神` 统一为 `用神`。来源由 ID 前缀区分：
- `zs-*` — 增删卜易
- `ym-*` — 易冒
- `gs-*` — 古筮真诠

### 2. 古筮真诠测例（首批）

新建 `references/古筮真诠.md`（已有，63021 行，1–40 章）。
按章节顺序从第 1 章到第 40 章逐一提取卦例。ID 格式：`gs-001`、`gs-002`……

### 3. 增删卜易测例补齐

在现有 zs-001~017 基础上，继续从 `references/增删卜易.md` 提取剩余卦例。
ID 格式：从 `zs-018` 起。

### 4. 易冒测例补齐

在现有 ym-001~008 基础上，继续从 `references/易冒.md` 提取剩余卦例。
ID 格式：从 `ym-009` 起。

### 5. JSON 字段约定

| 字段 | 必填 | 说明 |
|------|------|------|
| `id` | ✅ | 格式 `{前缀}-{三位序号}` |
| `来源` | ✅ | 含书名和章节，如"古筮真诠·第十一章" |
| `问题` | ✅ | 所占之事 |
| `时间` | ✅ | 干支时间 |
| `旬空` | ✅ | 可空（不知则填"未详"） |
| `本卦` | ✅ | 卦名 |
| `变卦` | ✅ | 无变动则填"无（静卦）" |
| `世应` | ✅ | 静卦可略 |
| `动爻` | ✅ | 静卦可略 |
| `日月建` | ✅ | |
| `六冲` | ✅ | true/false |
| `作者` | ✅ | 野鹤老人 / 程良玉 / 朱辰彬 |
| `断语` | ✅ | 原文断语（统一字段名） |
| `用神` | ✅ | 原文用神取法（统一字段名） |
| `断语吉凶` | ✅ | 吉/凶 |
| `原文关键分析` | ✅ | 推理逻辑摘要 |
| `应期` | ✅ | 可空 |
| `验证点` | ✅ | 字符串数组，每个对应 SKILL.md 中的关键概念 |

### 6. 提取顺序

1. 古筮真诠：第 1 章 → 第 40 章
2. 增删卜易：从 zs-018 起补齐
3. 易冒：从 ym-009 起补齐

### 7. 测试脚本更新

`scripts/verify-case-studies.sh` 需修改：

- ID 前缀检查：增加 `gs-*` 分支
- 来源统计：增加古筮真诠统计项
- 字段检查：`野鹤断语`/`作者断语` → `断语`，`野鹤用神`/`作者用神` → `用神`
- 最低数量门槛：从 25 调整为适当值（或随测例增长动态）
