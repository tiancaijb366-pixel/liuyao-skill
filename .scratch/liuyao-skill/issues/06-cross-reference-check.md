# 06 — 卦例间交叉引用检查

**What to build:** 建立测例间的交叉引用完整性检查机制，确保验证点词汇一致、概念覆盖完备、引用关系有效。

**Blocked by:** 05 — 测例扩充（需在全部测例就绪后检查交叉引用完整性）

**Status:** ready-for-agent

## Problem Statement

随着测例数量从 25 个扩充到上百个，以下问题会逐渐显现：
- 同一个概念在不同测例的 `验证点` 中用词不一致（如"六冲断法" vs "六冲判断"）
- 某概念在 SKILL.md 中有对应章节，但没有任何测例覆盖
- 某测例的 `原文关键分析` 引用另一个测例的 ID，但该 ID 不存在
- 无法直观知道哪些概念已有测例覆盖、哪些还缺

## Solution

新增一个验证脚本 `scripts/verify-cross-references.sh`，在 `verify-case-studies.sh` 之外独立运行，专门做测例间的关联性检查。

## 具体内容

### 1. 验证点词汇归一检查

收集全部测例的 `验证点` 数组，统计每个用词的出现次数。
- 识别拼写变体（如"六冲断法" vs "六冲判断"）
- 输出完整词汇表供人工审核
- 标记出现次数 ≤ 1 的冷僻词

### 2. 概念覆盖缺口分析

对照 SKILL.md 中定义的关键概念清单，检查：
- 哪些概念在至少一个测例的 `验证点` 中出现
- 哪些概念无任何测例覆盖（覆盖率缺口）
- 输出覆盖率百分比（在现有 83% 基础上继续追踪）

### 3. ID 引用完整性

扫描全部测例的 `原文关键分析` 字段，检测：
- 是否引用了其他测例 ID（如 `zs-003`、`gs-015`）
- 被引用的 ID 是否真实存在
- 标记悬挂引用（引用不存在 ID）

### 4. 输出格式

脚本输出三部分：

```
═══ 交叉引用检查 ═══

▸ 验证点词汇表
  ✅ 总词汇数: N
  ⚠️ 低频词（<=1次）: 词A, 词B ...
  ⚠️ 疑似变体: "六冲断法" vs "六冲判断"

▸ 概念覆盖
  ✅ SKILL.md 概念总数: N
  ✅ 有测例覆盖: N (X%)
  ❌ 无覆盖概念: 词A, 词B ...

▸ ID 引用完整性
  ✅ 引用总数: N
  ✅ 有效引用: N
  ❌ 悬挂引用: ID-A, ID-B ...
```

### 5. 与现有测试的关系

`verify-cross-references.sh` 作为独立脚本，不与 `verify-case-studies.sh` 合并。覆盖缺口是信息性报告而不是 pass/fail 检查，不会阻塞 CI。

## 验收标准

- [ ] `scripts/verify-cross-references.sh` 已创建
- [ ] 验证点词汇归一检查可用
- [ ] 概念覆盖缺口分析可用
- [ ] ID 引用完整性检查可用
- [ ] 对当前 25 个测例运行无报错（或报告已知缺口）
